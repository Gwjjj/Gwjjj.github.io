Session

概念

  1. 一次会话包含多次请求和响应
  2. 功能：在一次的会话范围内共享数据
  3. 方式
    1. 客户端会话技术：Cookie
    2. 服务器端会话技术：Session

cookie

  • 概念：客户端会话技术，将数据保存在客户端
  • 步骤
    1. 创建cookie，绑定数据
      new Cookie(String name, String value)
    2. 发送cookoe
      response.addCookie(Cookie cookie)
    3. 获取cookie，拿到数据
      request.getCookies()
  • 原理
    • 基于响应头set-cookie和请求头cookie实现
  • 持久化储存
    • setMaxAge(int seconds)
      1. 正数：写到硬盘的文江中
      2. 默认值：在浏览器内存里
      3. 0：删除cookie信息
  • 共享
    • setPath(String path)
    • 同一个服务器 setPath(“/“)
    • 不同服务器 setDomain(String path) # setDomain(“.baidu.com”) 一级域名
  • 特点
    1. cookie储存在客户端浏览器
    2. 浏览器对于单个cookie的大小有限制 以及同一个域名下的总cookie数量有限制

Session

  • 概念：服务器端会话技术，在一次会话的多次请求间共享数据，将数据保存在服务器端的对象中
  • HttpSession对象
    • Object getAttribute(String name)
    • void setAttribute(String name, Object value)
    • void removeAttribute(String name)
  • Session的实现是依赖于Cookie的
  • 客户端关闭保证session同一个：使用new Cookie(“JSESSIONID, session.getId())并持久化保存
  • 服务器关闭保证数据不丢失
    • session的钝化：在服务器正常关闭之前，将session对象序列化到硬盘上
    • session的活化：在服务器启动后将session文件转化为内存中的session对象
