Servlet

概念

  • server applet: 运行在服务器端的小程序
  • 一个接口，Java类被浏览器访问的规则
  • 版本
    • 3.0之前：web.xml
    • 3.0之后：注解 @WebServlet

生命周期

  1. 初始化 init
  2. 服务 service
  3. 销毁 destroy

    体系结构

    Servlet -> 抽象类GenericServlet -> 抽象类HttpServlet

  • GenericServlet将除了service方法的四个方法作了默认空实现，只需复写service方法

  • HttpServlet

    • 对http协议的一种封装

    • service

      1. 判断请求方式

      2. 获取数据
        doGet()
        doPost()
        其他五种

    • 使用

      1. 定义类继承HttpServlet
      2. 复写doGet()或者doPost()

    • 相关配置

      1. urlpattern：Servlet访问的路径
      2. 路径定义规则
        1. /xxx
        2. /xxx/xxx
        3. *.do

          HTTP

  • 超文本传输协议，客户端和服务器通信时发送数据的格式

  • 特点

    1. 基于TCP/IP的高级协议
    2. 默认端口号 80
    3. 基于请求/相应模型：一次请求对应一次响应
    4. 无状态：每次请求之间相互独立，不能交互数据

  • 历史版本

    • 1.0 每次请求响应都会建立新的连接
    • 1.1 会复用连接

  • 请求消息数据格式

    1. 请求行
      • 请求方式 请求url 请求协议/版本
      • GET /login.html HTTP/1.1
    2. 请求头
      • 请求头名称: 请求头值
      • User-Agent：浏览器版本信息
      • Referer：请求来源
        1. 防盗链
        2. 统计工作
    3. 请求空行
      空行：分割请求头和请求体
    4. 请求体(正文)
      post的键值对

Request

  • 作用

    • Request和Response都是由服务器创建的
    • Request对象是来获取请求消息，Response对象是来设置响应消息

  • Request对象继承体系结构

    • ServletRequest
      | 继承
    • HttpServletRequest
      | 实现
    • org.apach.catalina.connector.RequestFacade

  • Request功能

    1. 获取请求消息数据

      1. 获取请求行数据
        • 获取请求方式 getMethod
        • 获取虚拟目录 getContextPath
        • 获取Servlet路径 getServletPath
        • 获取get方式携带参数 getQueryString
        • 获取请求URI getRequestURI
        • 获取请求URL getRequestURL
        • 获取协议版本 getProtocol
        • 获取客户机的IP地址 getRemoteAddr
      2. 获取请求头数据
        • 通过请求头名称获取请求头值 getHeader
        • 获取所有请求头名称 getHeaderNames
      3. 获取请求体数据
        • 获取流对象(字符字节) getReader getInputStream
        • 从流对象中拿数据

    2. 其他功能

      1. 获取请求参数通用方式

      2. 请求转发: 一种在服务器内部的资源跳转方式

        • 步骤
          1. 通过request对象获取请求转发器对象：RequestDispatcher getRequestDispatcher(String path)
          2. 使用RequestDispatcher对象来进行转发: forward(ServletRequest request, ServletResponse response)
        • 特点
          1. 浏览器地址不会发生变化
          2. 只能转发到当前服务器内部资源中
          3. 转发是一次请求(可以共享数据)

      3. 共享数据

        • 域对象：一个有作用范围的对象，可以在范围内共享数据
        • request域：代表一次请求的范围，一般用于请求转发的多个资源中共享数据
        • 方法
          1. setAttribute(String name, Object obj) 存储数据
          2. Object getAttribute(String name) 获取值
          3. removeAttribute(String name) 移除数据

      4. 获取ServletContext

        • ServletContext getServletContext()

Response

  • 响应消息数据格式
    1. 响应行 HTTP/1.1 200 OK
      • 协议和版本
      • 响应状态码
        • 1XX 服务器接收客户端消息但没有接收完成
        • 2XX 成功
        • 3XX 重定向，访问缓存
        • 4XX 客户端错误
        • 5XX 服务器端错误
      • 状态码描述
    2. 响应头
      • 格式：头 值
      • 常见响应头
        1. Content-Type: 服务器告诉客户端本次响应体数据格式以及编码格式
        2. Content-disposition：服务器告诉客户端以什么格式打开相应体数据
          • in-line 在当前页面内打开(打开)
          • attachment;filename=xxx 以附件形式打开(文件下载)
    3. 响应空行
    4. 响应体
      • 传输的数据
  • Response对象
    • 功能：设置响应消息
      • 设置响应行
        1. 设置状态码 setStatus(int sc)
      • 设置响应头
        1. setHeader(String name, String value)
          • response.setStatus(302)
            response.setHeader(“location”,url) # 重定向的一种写法
      • 设置响应体
        1. 获取输出流
          • PrintWriter getWriter()
          • ServletOutputStream getOutputStream()
        2. 使用输出流
          • 字符输出流
          • 字节输出流
      • 重定向
        • response.sendRedirect(String s)
        • 特点
          • 地址栏变化
          • 可以访问其他站点的资源
          • 重定向是两次请求(不可以共享数据)
        • 路径写法
