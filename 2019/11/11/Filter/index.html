Filter

概念

  • web中的过滤器，当访问服务器的资源时可以将请求拦截下来，完成一些特殊的功能
  • 一般用于完成通用的操作。如登陆验证、统一编码处敏感字符过滤

方法

  1. 定义一个方法，实现接口Filter

  2. 复写方法

  3. 配置拦截路径

    1. web.xml
    2. 注解@(WebFilter(uri))
    3. doFilter(req,resp)

  4. 拦截路径配置

    • 具体资源路径：/index.jsp
    • 拦截目录：/user/*
    • 后缀名拦截：*.jsp
    • 拦截所有资源：/*

  5. 拦截方式配置

    • 注解配置 设置dispatcherTypes
      1. REQUEST 默认值 浏览器直接请求资源
      2. FORWARD 转发访问资源
      3. INCLUDE 包含访问资源
      4. ERROR 错误跳转
      5. ASYNC 异步资源
    • web.xml

  6. 过滤器链(配置多个过滤器)

    • 注解：按照类名的字符串规则比较，值小的先执行

    • web.xml：谁定义在上边，谁先执行
