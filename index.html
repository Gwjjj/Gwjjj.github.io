Session

概念

  1. 一次会话包含多次请求和响应
  2. 功能：在一次的会话范围内共享数据
  3. 方式
    1. 客户端会话技术：Cookie
    2. 服务器端会话技术：Session

cookie

  • 概念：客户端会话技术，将数据保存在客户端
  • 步骤
    1. 创建cookie，绑定数据
      new Cookie(String name, String value)
    2. 发送cookoe
      response.addCookie(Cookie cookie)
    3. 获取cookie，拿到数据
      request.getCookies()
  • 原理
    • 基于响应头set-cookie和请求头cookie实现
  • 持久化储存
    • setMaxAge(int seconds)
      1. 正数：写到硬盘的文江中
      2. 默认值：在浏览器内存里
      3. 0：删除cookie信息
  • 共享
    • setPath(String path)
    • 同一个服务器 setPath(“/“)
    • 不同服务器 setDomain(String path) # setDomain(“.baidu.com”) 一级域名
  • 特点
    1. cookie储存在客户端浏览器
    2. 浏览器对于单个cookie的大小有限制 以及同一个域名下的总cookie数量有限制

Session

  • 概念：服务器端会话技术，在一次会话的多次请求间共享数据，将数据保存在服务器端的对象中
  • HttpSession对象
    • Object getAttribute(String name)
    • void setAttribute(String name, Object value)
    • void removeAttribute(String name)
  • Session的实现是依赖于Cookie的
  • 客户端关闭保证session同一个：使用new Cookie(“JSESSIONID, session.getId())并持久化保存
  • 服务器关闭保证数据不丢失
    • session的钝化：在服务器正常关闭之前，将session对象序列化到硬盘上
    • session的活化：在服务器启动后将session文件转化为内存中的session对象

Servlet

概念

  • server applet: 运行在服务器端的小程序
  • 一个接口，Java类被浏览器访问的规则
  • 版本
    • 3.0之前：web.xml
    • 3.0之后：注解 @WebServlet

生命周期

  1. 初始化 init
  2. 服务 service
  3. 销毁 destroy

    体系结构

    Servlet -> 抽象类GenericServlet -> 抽象类HttpServlet

  • GenericServlet将除了service方法的四个方法作了默认空实现，只需复写service方法

  • HttpServlet

    • 对http协议的一种封装

    • service

      1. 判断请求方式

      2. 获取数据
        doGet()
        doPost()
        其他五种

    • 使用

      1. 定义类继承HttpServlet
      2. 复写doGet()或者doPost()

    • 相关配置

      1. urlpattern：Servlet访问的路径
      2. 路径定义规则
        1. /xxx
        2. /xxx/xxx
        3. *.do

          HTTP

  • 超文本传输协议，客户端和服务器通信时发送数据的格式

  • 特点

    1. 基于TCP/IP的高级协议
    2. 默认端口号 80
    3. 基于请求/相应模型：一次请求对应一次响应
    4. 无状态：每次请求之间相互独立，不能交互数据

  • 历史版本

    • 1.0 每次请求响应都会建立新的连接
    • 1.1 会复用连接

  • 请求消息数据格式

    1. 请求行
      • 请求方式 请求url 请求协议/版本
      • GET /login.html HTTP/1.1
    2. 请求头
      • 请求头名称: 请求头值
      • User-Agent：浏览器版本信息
      • Referer：请求来源
        1. 防盗链
        2. 统计工作
    3. 请求空行
      空行：分割请求头和请求体
    4. 请求体(正文)
      post的键值对

Request

  • 作用

    • Request和Response都是由服务器创建的
    • Request对象是来获取请求消息，Response对象是来设置响应消息

  • Request对象继承体系结构

    • ServletRequest
      | 继承
    • HttpServletRequest
      | 实现
    • org.apach.catalina.connector.RequestFacade

  • Request功能

    1. 获取请求消息数据

      1. 获取请求行数据
        • 获取请求方式 getMethod
        • 获取虚拟目录 getContextPath
        • 获取Servlet路径 getServletPath
        • 获取get方式携带参数 getQueryString
        • 获取请求URI getRequestURI
        • 获取请求URL getRequestURL
        • 获取协议版本 getProtocol
        • 获取客户机的IP地址 getRemoteAddr
      2. 获取请求头数据
        • 通过请求头名称获取请求头值 getHeader
        • 获取所有请求头名称 getHeaderNames
      3. 获取请求体数据
        • 获取流对象(字符字节) getReader getInputStream
        • 从流对象中拿数据

    2. 其他功能

      1. 获取请求参数通用方式

      2. 请求转发: 一种在服务器内部的资源跳转方式

        • 步骤
          1. 通过request对象获取请求转发器对象：RequestDispatcher getRequestDispatcher(String path)
          2. 使用RequestDispatcher对象来进行转发: forward(ServletRequest request, ServletResponse response)
        • 特点
          1. 浏览器地址不会发生变化
          2. 只能转发到当前服务器内部资源中
          3. 转发是一次请求(可以共享数据)

      3. 共享数据

        • 域对象：一个有作用范围的对象，可以在范围内共享数据
        • request域：代表一次请求的范围，一般用于请求转发的多个资源中共享数据
        • 方法
          1. setAttribute(String name, Object obj) 存储数据
          2. Object getAttribute(String name) 获取值
          3. removeAttribute(String name) 移除数据

      4. 获取ServletContext

        • ServletContext getServletContext()

Response

  • 响应消息数据格式
    1. 响应行 HTTP/1.1 200 OK
      • 协议和版本
      • 响应状态码
        • 1XX 服务器接收客户端消息但没有接收完成
        • 2XX 成功
        • 3XX 重定向，访问缓存
        • 4XX 客户端错误
        • 5XX 服务器端错误
      • 状态码描述
    2. 响应头
      • 格式：头 值
      • 常见响应头
        1. Content-Type: 服务器告诉客户端本次响应体数据格式以及编码格式
        2. Content-disposition：服务器告诉客户端以什么格式打开相应体数据
          • in-line 在当前页面内打开(打开)
          • attachment;filename=xxx 以附件形式打开(文件下载)
    3. 响应空行
    4. 响应体
      • 传输的数据
  • Response对象
    • 功能：设置响应消息
      • 设置响应行
        1. 设置状态码 setStatus(int sc)
      • 设置响应头
        1. setHeader(String name, String value)
          • response.setStatus(302)
            response.setHeader(“location”,url) # 重定向的一种写法
      • 设置响应体
        1. 获取输出流
          • PrintWriter getWriter()
          • ServletOutputStream getOutputStream()
        2. 使用输出流
          • 字符输出流
          • 字节输出流
      • 重定向
        • response.sendRedirect(String s)
        • 特点
          • 地址栏变化
          • 可以访问其他站点的资源
          • 重定向是两次请求(不可以共享数据)
        • 路径写法

Maven

Maven 是一个项目管理工具，可以对 Java 项目进行构建、依赖管理。

仓库
  1. 本地仓库
  2. 远程仓库(私服)
  3. 中央仓库
    项目目录
  4. src/main/java 核心代码部分
  5. src/main/resoucres 配置文件部分
  6. src/test/java 测试代码部分
  7. src/test/resources 测试配置文件
  8. src/main/webapp 页面资源，js，css，图片等

  12. mvn test(test文件下会被编译)

  14. mvn install(把包安装到本地仓库)

格式

  1. 项目自身信息

  2. 项目运行所依赖的jar包信息

  3. 项目运行环境信息

JQuery

概念

一个javaScript的框架。简化js的开发

转换

  1. js -> jq: $(js对象)
  2. jq -> js: jq对象[index]或者jq对象.get(index)

    选择器

  3. 入口函数
    $(function(){})相当于windows.onload # onload只能执行一次
  4. 基本选择器
    1. 标签选择器
      • $(“html标签名”)
    2. id选择器
      • $(“#id”)
    3. 类选择器
      • $(“.class属性”)
    4. 并集选择器
      • $(“选择器1,选择器2”)
  5. 层级选择器
    1. 后代选择器
      • $(“A B”) A元素下的所有B元素
    2. 子选择器
      • $(“A > B”) A元素下的所有B子元素
  6. 属性选择器
    1. 属性名称选择器
      • $(“A[属性名]”)
    2. 属性选择器
      • $(“A[属性名=’值’]”)
    3. 符合属性选择器
      • $(“A[属性名=’值’][]…”)
  7. 过滤选择器
    1. 首元素选择器
      • :first
    2. 尾元素选择器
      • :last
    3. 非元素选择器
      • :not(selecter) 不包括指定内容的元素
    4. 偶数选择器
      • :even 偶数,从0开始计数
    5. 奇数选择器
      • :odd 奇数，从0开始计数
    6. 等于索引选择器
      • :eq(index) 指定索引元素
    7. 大于索引选择器
      • :gt(index)
    8. 小于索引选择器
      • :lt(index)
    9. 标题选择器
      • :header 获得标题(h1~h6)元素
  8. 表单过滤选择器
    1. 可用元素选择器
      • :enabled 获得可用元素
    2. 不可用元素选择器
      • :disabled 获得不可用元素
    3. 选中选择器
      • :checked 获得单选/复选框选中的元素
    4. 选中选择器
      • :selected 获得下拉框选中的元素

        DOM操作

  9. 内容操作
    1. html() 标签体内容(包含子标签体)
    2. text() 标签体纯文本内容
    3. val() 获取/设置元素的value属性值
  10. 属性操作
    1. 通用属性操作
      1. attr(name,val)
      2. removeAttr(name)
      3. prop(name,val)
      4. removeProp(name)
      5. css(属性,值)

        区别：如果操作是元素的固有属性，则建议使用prop。自定义属性则用attr。

    2. 对class属性操作
      1. addClass(class) 与prop(“class”,class)类似
      2. removeClass(class)
      3. toggleClass(class) 切换class(如果存在(不存在)就删除(添加)一个类)
  11. CRUD操作
    1. append() 父元素将子元素添加到末尾
      对象一.append(对象二): 将对象二添加到对象一元素内部，并且在末尾
    2. prepend() 在开头
    3. appendTo() 与1相反
    4. prependTo() 与2相反
    5. after() 添加元素到元素后边(兄弟关系)
      对象一.after(对象二) 将对象二添加到对象一后边
    6. before() 前边
    7. insertAfter() 与5相反
    8. insertBefore() 与6相反
    9. remove() 移除元素
    10. empty() 清空所有后代元素

高级

  1. 动画
    1. 默认显示和隐藏方式
      1. show([speed,[easing],[fn]])
      2. hide([speed,[easing],[fn]])
      3. toggle([speed,[easing],[fn]])
    2. 滑动显示和隐藏方式
      1. slideDown([speed,[easing],[fn]])
      2. slideUp([speed,[easing],[fn]])
      3. slideToggle([speed,[easing],[fn]])
    3. 淡入淡出
      1. fadeIn([speed,[easing],[fn]])
      2. fadeOut([speed,[easing],[fn]])
      3. fadeToggle([speed,[easing],[fn]])
    4. 参数
      1. speed 动画速度 (slow, normal, fast)或者动画时长毫秒值
      2. easing 切换效果 (swing, liner)(swing先慢再快再慢,liner匀速)
      3. fn 再动画完成时执行的函数，每个元素执行一次。
  2. 遍历
    1. jq对象.each(function(index, elemenvt))
    2. $.each(Object, callback )
    3. for..of: 3.0之后提供
    4. return false/return true = break/continue
  3. 事件绑定
    1. jquery标准的绑定方式
      js对象.事件方法(callback)
    2. on绑定事件/off解除绑定
      jq对象.on(“事件名称”,callback)
      jq对象.off(“事件名称”)
    3. 事件切换：toggle
      jq对象.toggle(fn1,fn2…)
  4. 插件
  • 增强JQuery的功能
  • $.fn.extend(object) 添加jQuery功能
    • $.fn.extend({check:callback})
  • $.extend(object) 增强jQuery对象自身的功能
    • $/JQuery

Filter

概念

  • web中的过滤器，当访问服务器的资源时可以将请求拦截下来，完成一些特殊的功能
  • 一般用于完成通用的操作。如登陆验证、统一编码处敏感字符过滤

方法

  1. 定义一个方法，实现接口Filter

  2. 复写方法

  3. 配置拦截路径

    1. web.xml
    2. 注解@(WebFilter(uri))
    3. doFilter(req,resp)

  4. 拦截路径配置

    • 具体资源路径：/index.jsp
    • 拦截目录：/user/*
    • 后缀名拦截：*.jsp
    • 拦截所有资源：/*

  5. 拦截方式配置

    • 注解配置 设置dispatcherTypes
      1. REQUEST 默认值 浏览器直接请求资源
      2. FORWARD 转发访问资源
      3. INCLUDE 包含访问资源
      4. ERROR 错误跳转
      5. ASYNC 异步资源
    • web.xml

  6. 过滤器链(配置多个过滤器)

    • 注解：按照类名的字符串规则比较，值小的先执行

    • web.xml：谁定义在上边，谁先执行

AJAX

概念

ASynchronous JavaScript And XML 异步的js和xml
无需加载整个页面的情况下能够更新部分网页技术

实现方式

  1. 异步和同步
    客户端和服务器相互通信的基础上
    同步：客户端必须等待服务器的响应，等待期间不能做其他操作
    异步：客户端不需要等待服务器的响应

  2. 实现方式

    1. 原生的js实现方式

    2. JQuery实现方式

      1. $.ajax()

      2. $.get()

      3. $.post()
